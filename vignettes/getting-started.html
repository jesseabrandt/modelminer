Getting Started with modelminer

Overview

modelminer automates forward model building by iteratively adding terms — polynomial expansions and interaction terms — that most improve a model selection metric. You supply a model function, a dataset, and a response variable; mine() explores the candidate term space and returns the best formula it finds.

The default metric is AIC and the default search is greedy forward selection, but both are configurable. mine() accepts any model function with a formula and data interface, which covers lm, glm, and anything wrapped with formula_wrap().

Example 1: Linear model

The simplest case — finding predictors of fuel economy in mtcars:

library(modelminer)

lm_result <- mine(mtcars, mpg, max_degree = 2, max_interact_vars = 2)
lm_result$Formula
#> mpg ~ 1 + cyl:wt + vs:am + hp:qsec + I(hp^2)
#> <environment: 0xaaab13865d00>

The all_models data frame records every formula evaluated during the search, including the metric value at each step. This is useful for inspecting the search path, diagnosing whether the algorithm stopped too early, or comparing how metric values evolved:

head(lm_result$all_models[, "Formula", drop = FALSE])
#>          Formula
#> 1        mpg ~ 1
#> 2  mpg ~ 1 + cyl
#> 3 mpg ~ 1 + disp
#> 4   mpg ~ 1 + hp
#> 5 mpg ~ 1 + drat
#> 6   mpg ~ 1 + wt
nrow(lm_result$all_models)
#> [1] 316

Example 2: Binomial GLM with an anonymous function

mine() is not limited to lm. Any model function with a formula and data interface works. The model_func argument accepts anonymous functions, which is the natural way to fix extra arguments (like family) while keeping the signature compatible with what mine() expects.

Here we search for predictors of transmission type in mtcars (am: 0 = automatic, 1 = manual):

glm_result <- mine(
  mtcars, am,
  model_func        = \(formula, data) glm(formula, data, family = binomial),
  max_degree        = 1,
  max_interact_vars = 1
)

Example 3: Matrix-based models with formula_wrap()

Some model functions don’t accept a formula — glmnet::cv.glmnet takes a predictor matrix x and response vector y instead. formula_wrap() adapts such functions to mine()’s interface by extracting x and y from a data frame using the formula.

The anonymous function pattern from Example 2 applies here too: use it to fix arguments the wrapper can’t infer, such as the family.

# Requires the glmnet package.
# install.packages("glmnet")

formula_cv_glmnet <- formula_wrap(
  \(x, y, ...) glmnet::cv.glmnet(x, y, ...)
)

# cv.glmnet needs at least one predictor column, so keep_all_vars = TRUE
# avoids passing an empty matrix for the intercept-only starting model.
glmnet_result <- mine(
  mtcars, mpg,
  model_func        = formula_cv_glmnet,
  metric            = \(m) min(m$cvm),   # minimum cross-validated MSE
  metric_comparison = min,
  keep_all_vars     = TRUE,
  max_degree        = 1,
  max_interact_vars = 1
)

glmnet_result$Formula

formula_wrap() assumes the wrapped function uses x and y as argument names (the glmnet convention). If your function uses different names, pass them as x_name and y_name:

# Example for a hypothetical function that uses "predictors" and "outcome"
formula_custom <- formula_wrap(my_model_func, x_name = "predictors", y_name = "outcome")

Comparing configurations with compare_methods()

When experimenting with search settings — different algorithms, polynomial degrees, or interaction depth — compare_methods() runs multiple configurations and collects results in one call:

cmp <- compare_methods(
  mtcars, mpg,
  configs = list(
    greedy_linear   = list(method = "greedy",          max_degree = 1),
    greedy_poly     = list(method = "greedy",          max_degree = 3),
    forward_backward = list(method = "forward_backward", max_degree = 1)
  ),
  max_interact_vars = 1
)
cmp$summary[, c("Config", "Formula", "BestMetric", "ModelsEvaluated")]
#>             Config                             Formula BestMetric
#> 1    greedy_linear             mpg ~ 1 + wt + cyl + hp   155.4766
#> 2      greedy_poly mpg ~ 1 + wt + cyl + I(wt^2) + qsec   148.4657
#> 3 forward_backward                 mpg ~ wt + cyl + hp   155.4766
#>   ModelsEvaluated
#> 1              35
#> 2             141
#> 3              44

The details element of the result holds the full mine() output for each configuration, so you can inspect all_models for any of them individually.
